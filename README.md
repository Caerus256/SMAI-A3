# Regional Thali Identifier

**SMAI Assignment 3 - Task T2.6**

Indian food image classifier (80 classes). Upload a photo of a dish and the app predicts the dish name, maps it to a region (North / South / East / West / North East), estimates calories, and flags common allergens (nuts, dairy, gluten).

**Live Demo:** https://smai-a3-jmhqgx2jwpsq2bb6ypwqpa.streamlit.app/

**GitHub Repository:** https://github.com/Caerus256/SMAI-A3

## Setup

```bash
git clone https://github.com/Caerus256/SMAI-A3.git
cd T2.6-Regional-Thali-Identifier
pip install -r requirements.txt
```

Place the `Indian Food Images/` folder (80 subfolders, 50 images each) and `indian_food.csv` inside this directory. Both are available from Kaggle (links at the bottom).

## How to Run

### 1. Generate metadata (only needed once)

```bash
python data_audit.py
```

This reads `indian_food.csv` and the image directory, filters to the 80 classes that have images, and produces two files:

- `data/food_metadata_80.csv` — per-dish metadata (name, region, state, ingredients, diet, course, flavor, prep/cook time)
- `data/nutrition_allergens.json` — per-dish calorie estimates, allergen flags (nuts/dairy/gluten), and serving sizes


### 2. Train models

Open `training.ipynb` and run all cells. The notebook contains:

| Section | What it does |
|---|---|
| Setup + Imports | Loads metadata, configures paths and plotting |
| 1. EDA | Region, diet, course, and flavor distribution charts |
| 2. Simple CNN | Trains a CNN from scratch (baseline, ~10% accuracy) |
| 3. CLIP Zero-Shot | Zero-shot classification with CLIP ViT-B/32 (~45%) |
| 4. CLIP Linear Probe | Extracts CLIP features, trains sklearn LogisticRegression (~75%, best model) |
| 5. EfficientNet-B0 | Head-only and last-block fine-tuning at 5 and 10 epochs |
| 6. EfficientNet-B3 | Progressive unfreezing with test-time augmentation |
| 7. ViT-B/16 | Head-only and last-2-blocks fine-tuning at 5 and 10 epochs |
| 8. Ablation | Comparison table and bar chart of all configurations |
| 9. Sample predictions | Visual grid of predictions on test images |

Outputs:
- Model weights in `model/` (`.pth` for PyTorch models, `.pkl` for the CLIP linear probe)
- Figures in `figures/`
- `data/ablation_results.csv`

GPU is recommended for training. Total time is roughly 1-2 hours on a modern GPU.

### 3. Launch the Streamlit app

```bash
streamlit run app.py
```

The app starts at `http://localhost:8501`. You can also use the deployed version linked above.

The app lets you:
- Upload a food photo
- Select a model (CLIP, ViT, or EfficientNet) from the sidebar
- Get predictions: dish name, confidence, region, state, calories, diet, course, flavor, prep/cook time, allergen flags, top-3 alternatives, and ingredients

Model weights are auto-downloaded from HuggingFace Hub (`Caerus256/SMAI-A3`) if not found locally.

### 4. Docker (optional)

```bash
docker build -t regional-thali-identifier .
docker run -p 8501:8501 regional-thali-identifier
```

The Dockerfile uses Python 3.10-slim and exposes port 8501 for Streamlit.

## File Descriptions

### Core files

- **`app.py`** — Streamlit web app. Loads models (CLIP / ViT / EfficientNet), runs inference on uploaded images, displays prediction results. Supports auto-downloading weights from HuggingFace Hub if they are missing locally.

- **`training.ipynb`** — Jupyter notebook with the full training pipeline: EDA, data loading, training loops for all model architectures, evaluation metrics, ablation comparison, and visualization. Run all cells top to bottom to reproduce.

- **`data_audit.py`** — Reads `indian_food.csv` and matches entries against the 80 image class folders. Outputs `food_metadata_80.csv` and `nutrition_allergens.json`. Uses keyword lists for allergen detection and a heuristic for calorie estimation.

- **`requirements.txt`** — Python dependencies: torch, torchvision, timm, open_clip_torch, streamlit, Pillow, pandas, numpy, matplotlib, seaborn, scikit-learn, tqdm, huggingface_hub.

### utils/

- **`data_utils.py`** — Helper functions: `load_metadata()`, `load_nutrition()`, `get_class_mappings()` (class-to-index, index-to-class, dish-to-region, region-to-index mappings), and `get_inference_transform()`. Defines constants like `NUM_CLASSES = 80`, `REGION_ORDER`, and ImageNet normalization values.

- **`predict.py`** — Standalone inference helper. `load_model()` loads an EfficientNet-B0 checkpoint; `predict_image()` takes a PIL image and returns top-k predictions with all metadata. Useful for batch inference outside the web app.

### data/

- **`food_metadata_80.csv`** — 80 rows, columns: folder_name, display_name, ingredients, diet, course, flavor_profile, state, region, prep_time, cook_time.

- **`nutrition_allergens.json`** — JSON dict keyed by folder_name. Each entry has: display_name, allergens (nuts/dairy/gluten booleans), approx_calories_kcal, diet, course, region, state, flavor_profile, ingredients, prep_time_min, cook_time_min, serving_size.

- **`ablation_results.csv`** — 15 rows with columns: Model, Training, Dish Top-1, Dish Top-3, Dish F1, Region Acc, Region F1.

### model/

Trained model weights. `.pth` and `.pkl` files are tracked with Git LFS (see `.gitattributes`). Also excluded via `.gitignore` since they can be auto-downloaded at runtime.

| File | Size | Description |
|---|---|---|
| clip_linear_probe.pkl | ~330 KB | sklearn LogisticRegression on CLIP ViT-B/32 features |
| vit_b16_best.pth | ~343 MB | ViT-B/16, last-2-blocks fine-tuned |
| vit_b16_head_only.pth | ~343 MB | ViT-B/16, head-only |
| efficientnet_b0_best.pth | ~16 MB | EfficientNet-B0, last-block fine-tuned |
| efficientnet_b0_head_only.pth | ~16 MB | EfficientNet-B0, head-only |
| efficientnet_b3_progressive.pth | ~44 MB | EfficientNet-B3, progressive unfreezing |

### figures/

All generated by the training notebook. Region distribution, sample mosaic, diet/course/flavor charts, ablation comparison bar chart, region confusion matrix, sample predictions grid, CNN training curves.

## Results

| Model | Training | Dish Top-1 | Dish Top-3 | Dish F1 | Region Acc | Region F1 |
|---|---|---|---|---|---|---|
| Simple CNN | From scratch, 10 ep | 10.17% | 22.00% | 6.56% | 38.67% | 30.50% |
| CLIP ViT-B/32 | Zero-shot | 45.33% | 58.17% | 41.62% | 60.50% | 48.89% |
| **CLIP ViT-B/32** | **Linear probe** | **74.83%** | **89.67%** | **73.53%** | **84.00%** | **79.21%** |
| EfficientNet-B0 | Head-only, 5 ep | 34.00% | 54.00% | 31.90% | 58.50% | 49.36% |
| EfficientNet-B0 | Last-block, 10 ep | 49.83% | 72.33% | 48.79% | 69.50% | 62.22% |
| EfficientNet-B3 | Progressive, 7 ep | 51.00% | 70.83% | 49.33% | 68.33% | 62.47% |
| EfficientNet-B3 + TTA | + TTA | 52.67% | 73.50% | 51.58% | 69.33% | 63.15% |
| Ensemble CLIP+B3 | 0.8/0.2 blend | 71.50% | 88.17% | 70.77% | 81.83% | 76.91% |
| ViT-B/16 | Head-only, 10 ep | 63.67% | 83.00% | 62.50% | 80.33% | 77.39% |
| ViT-B/16 | Last-2-blk, 5 ep | 69.33% | 84.83% | 69.20% | 83.17% | 80.86% |
| ViT-B/16 | Last-2-blk, 10 ep | 69.33% | 85.17% | 69.06% | 82.50% | 79.48% |

Full ablation in `data/ablation_results.csv`.

The CLIP linear probe is the best single model overall, outperforming all fine-tuned architectures using just a logistic regression on frozen CLIP features.
