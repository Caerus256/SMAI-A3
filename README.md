## Overview

This project builds an **80-class Indian food image classifier** that:

1. **Identifies the dish** from a photo (Top-1 and Top-3 accuracy).
2. **Maps the dish to a region** of India (North, South, East, West, North East).
3. **Estimates approximate calories** per serving.
4. **Flags allergens** — nuts, dairy, and gluten — based on ingredient keyword matching.

Multiple model architectures are trained and compared: a **Simple CNN** (from scratch), **CLIP ViT-B/32** (zero-shot + linear probe), **EfficientNet-B0/B3** (fine-tuned), and **ViT-B/16** (fine-tuned). The best model (CLIP Linear Probe) achieves **74.8% Top-1** dish accuracy and **84.0% region accuracy**.

A **Gradio web app** (`app.py`) lets users upload a photo and get instant predictions with all metadata.

## Features

- 🖼️ **Multi-model inference** — choose between CLIP, ViT, or EfficientNet at runtime.
- 🗺️ **Region mapping** — every dish is automatically mapped to one of five Indian regions.
- 🔥 **Calorie estimation** — heuristic calorie estimates based on course type and ingredients.
- 🚨 **Allergen detection** — keyword-based flagging of nuts, dairy, and gluten.
- 📊 **Comprehensive EDA** — region/diet/course/flavor distribution plots.
- 🐳 **Docker support** — one-command containerized deployment.
- 🤗 **HuggingFace Hub integration** — model weights auto-download from `Caerus256/SMAI-A3`.

## Data Preparation

Before training, run the data audit script **once** to generate the derived metadata files:

```bash
python data_audit.py
```

**What it does:**

1. Reads `indian_food.csv` (255 dishes) and the `Indian Food Images/` directory (80 classes).
2. Matches dishes that have both metadata **and** images → produces `data/food_metadata_80.csv` with columns: `folder_name`, `display_name`, `ingredients`, `diet`, `course`, `flavor_profile`, `state`, `region`, `prep_time`, `cook_time`.
3. For each dish, estimates **approximate calories** using a heuristic (base kcal per course ± ingredient modifiers ± jitter).
4. Detects **allergens** (nuts, dairy, gluten) by keyword matching against ingredient lists.
5. Writes `data/nutrition_allergens.json` with per-dish nutrition, allergens, and serving size.
6. Prints a region distribution summary and allergen statistics.

**Expected output:**

```
Found 80 image class folders
Matched 80 dishes to image folders
Wrote data/food_metadata_80.csv
Wrote data/nutrition_allergens.json

Region distribution (dishes / images):
  North: 32 dishes / 1600 images
  South: 15 dishes / 750 images
  West: 15 dishes / 750 images
  East: 14 dishes / 700 images
  North East: 4 dishes / 200 images

Allergen summary:
  Nuts:   XX/80 dishes
  Dairy:  XX/80 dishes
  Gluten: XX/80 dishes
```

## Training the Models

### Open the training notebook

```bash
jupyter notebook training.ipynb
```

Or use JupyterLab / VS Code / Google Colab.

### Notebook structure

The notebook is divided into the following sections:

| Section | Description |
|---|---|
| **Setup & Imports** | Loads metadata, sets paths, configures plotting |
| **1. EDA** | Region distribution, sample mosaic, diet/course/flavor charts |
| **2. Simple CNN** | Trains a CNN from scratch (baseline, ~10% acc) |
| **3. CLIP Zero-Shot** | Zero-shot classification with CLIP ViT-B/32 |
| **4. CLIP Linear Probe** | Extracts CLIP features → trains sklearn LogisticRegression (**best model**) |
| **5. EfficientNet-B0** | Head-only and last-block fine-tuning (5 & 10 epochs) |
| **6. EfficientNet-B3** | Progressive unfreezing with TTA |
| **7. ViT-B/16** | Head-only and last-2-blocks fine-tuning (5 & 10 epochs) |
| **8. Ablation & Comparison** | Summary table, bar chart, region confusion matrix |
| **9. Sample Predictions** | Visual grid of predictions on test images |

### How to run

1. **Ensure `data/food_metadata_80.csv` and `data/nutrition_allergens.json` exist** (run `data_audit.py` first).
2. **Run all cells** sequentially from top to bottom.
3. Training will produce:
   - Model weights in `model/` (`.pth` for PyTorch, `.pkl` for sklearn)
   - Figures in `figures/`
   - Ablation results in `data/ablation_results.csv`

### Training tips

- **GPU is strongly recommended** for EfficientNet and ViT training.
- CLIP feature extraction can run on CPU but is faster on GPU.
- The Simple CNN is intentionally weak (baseline) — expect ~10% accuracy.
- Total training time: ~1–2 hours on a modern GPU.

## Running the Web App

The app uses **Gradio** (not Streamlit — despite the `.streamlit/` config directory, the actual app is Gradio-based).

### Launch locally

```bash
python app.py
```

The app will start on `http://0.0.0.0:7860` by default.

### How it works

1. **Upload** a photo of an Indian dish.
2. **Select a model** from the dropdown (CLIP, ViT, or EfficientNet).
3. Click **🔍 Analyze**.
4. The app displays:
   - 🍛 **Dish name** and confidence score
   - 🗺️ **Region** and state of origin
   - 📊 **Calories** and serving size
   - 🥗 **Diet**, course, and flavor profile
   - ⏱️ Prep and cook time
   - 🚨 **Allergen flags** (nuts, dairy, gluten)
   - 🔮 **Top-3 alternative predictions**
   - 📝 **Ingredients list**

### Model auto-download

If model weights are not found in `model/`, the app will automatically attempt to download them from HuggingFace Hub (`Caerus256/SMAI-A3`). No manual download is needed if the HF repo is accessible.

### Model performance shown in the UI

## Ablation Study

A full ablation study is saved in `data/ablation_results.csv`. Key findings:

1. **From-scratch CNN** barely beats random (10% on 80 classes) — confirms the need for transfer learning on this small dataset.
2. **CLIP zero-shot** already reaches 45% Top-1 without any training.
3. **CLIP linear probe** is the overall winner at 74.8% — CLIP's pretrained representations are extremely powerful.
4. **ViT-B/16** last-2-blocks fine-tuning reaches 69.3%, the best among trained PyTorch models.
5. **EfficientNet-B3 + TTA** improves over B0 by ~3% but still trails CLIP and ViT significantly.
6. **Ensemble (CLIP + B3)** with 80/20 weighting reaches 71.5%, still below CLIP alone.

## Troubleshooting

| Issue | Solution |
|---|---|
| `FileNotFoundError: food_metadata_80.csv` | Run `python data_audit.py` first to generate metadata files. |
| `ModuleNotFoundError: open_clip` | Install with `pip install open_clip_torch`. The CLIP model is optional — ViT and EfficientNet still work. |
| Model weights not found | The app auto-downloads from HuggingFace Hub. Ensure internet access, or manually place weights in `model/`. |
| CUDA out of memory during training | Reduce batch size in the training notebook, or use CPU (slower). |
| Git LFS errors on clone | Install Git LFS (`git lfs install`) before cloning. Or simply let the app download weights at runtime. |
| Port conflict on 7860 | Change the port in `app.py` at the bottom: `demo.launch(server_port=XXXX)` |
